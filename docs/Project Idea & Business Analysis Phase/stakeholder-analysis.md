# Stakeholder Analysis

## Primary Stakeholders

### Students

Role:

* Main users of the system.

Interests:

* Easy assignment tracking.
* Timely reminders.
* User-friendly interface.

Benefits:

* Improved academic organization.
* Better deadline management.

## Secondary Stakeholders

### Faculty Members

Role:

* Assignment providers.

Interests:

* Students submitting assignments on time.

Benefits:

* Improved assignment completion rates.

### University Administration

Role:

* Potential adopters of the system.

Interests:

* Enhancing student productivity.

Benefits:

* Improved academic performance statistics.

## Development Team

### Developers

Role:

* Design, develop, and maintain the application.

Responsibilities:

* Frontend development.
* Backend development.
* Database management.
* Testing and deployment.
